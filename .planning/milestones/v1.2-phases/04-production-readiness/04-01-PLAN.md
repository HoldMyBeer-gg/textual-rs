  <name>Task 1: Add text selection to Input with Shift+key bindings and clipboard</name>

  <action>
Add selection state to Input:
- Add `selection_anchor: Cell<Option<usize>>` field (byte offset). When Some, text between anchor and cursor_pos is selected.
- Add `pub fn has_selection(&self) -> bool` method.
- Add `fn selected_range(&self) -> Option<(usize, usize)>` returning (start, end) byte offsets.
- Add `fn selected_text(&self) -> Option<String>` extracting the selected substring.
- Add `fn delete_selection(&self, ctx: &AppContext)` that removes selected text, updates cursor, clears anchor, emits Changed.

Add Shift+key bindings to INPUT_BINDINGS:
- Shift+Left: "select_left"
- Shift+Right: "select_right"
- Shift+Home: "select_home"
- Shift+End: "select_end"
- Ctrl+Shift+Left: "select_word_left"
- Ctrl+Shift+Right: "select_word_right"
- Ctrl+A: "select_all"
- Ctrl+C: "copy"
- Ctrl+X: "cut"
- Ctrl+V: "paste"

Implement on_action handlers:
- "select_left/right/home/end/word_left/word_right": Set anchor if None (to current cursor_pos), then move cursor. Selection = anchor..cursor.
- "select_all": Set anchor=0, cursor_pos=value.len().
- "copy": If has_selection, copy selected_text to arboard clipboard. Fail silently on clipboard error.
- "cut": Copy + delete_selection.
- "paste": If has_selection, delete_selection first, then insert clipboard text at cursor.

Modify existing movement actions (cursor_left, cursor_right, etc.): Clear selection_anchor when moving without Shift.

Modify on_event char insertion: If has_selection, delete_selection before inserting.

Update render(): Draw selected text with REVERSED modifier. The selection range maps to char indices; iterate display chars and apply REVERSED style for those in the selection range.

IMPORTANT: In on_event, when receiving Ctrl+C, check has_selection first. If selection exists, do copy and return Stop. The app.rs global Ctrl+C quit check must be modified (in Task 3).

<task type="auto">
  <name>Task 3: Route Ctrl+C to copy when text widget has selection (not quit)</name>
  <files>crates/textual-rs/src/app.rs, crates/textual-rs/src/widget/mod.rs</files>
  <action>
Add to Widget trait a default method:
```rust
fn has_text_selection(&self) -> bool { false }
```

Override in Input and TextArea to return `self.has_selection()`.

In app.rs event loop, modify the Ctrl+C quit check (around line 254-259). Before breaking on Ctrl+C:
1. Check if focused_widget exists and has_text_selection() returns true.
2. If so, dispatch the key event to the focused widget (which will handle it as "copy" via key_bindings) and do NOT break.
3. Only break on Ctrl+C if no widget has text selection, OR if the user presses Ctrl+C twice within 500ms (double-tap to quit pattern).

Implementation:
- Add a `last_ctrl_c: Option<std::time::Instant>` field to App.
- On Ctrl+C: if focused widget has selection, route to widget. If no selection, check if last_ctrl_c was <500ms ago — if so, break. Otherwise, set last_ctrl_c = now and continue (user gets one "grace" press).
- This prevents accidental quit when the user means to copy.
