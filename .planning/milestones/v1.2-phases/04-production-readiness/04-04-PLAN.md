**Animation module (new file animation.rs):**
Create a lightweight tweening system:

```rust
use std::time::{Duration, Instant};

pub type EasingFn = fn(f64) -> f64;

pub fn ease_in_out_cubic(t: f64) -> f64 {
    if t < 0.5 { 4.0 * t * t * t } else { 1.0 - (-2.0 * t + 2.0).powi(3) / 2.0 }
}

pub fn linear(t: f64) -> f64 { t }

pub struct Tween {
    pub from: f64,
    pub to: f64,
    pub duration: Duration,
    pub easing: EasingFn,
    pub start_time: Instant,
}

impl Tween {
    pub fn new(from: f64, to: f64, duration: Duration, easing: EasingFn) -> Self {
        Self { from, to, duration, easing, start_time: Instant::now() }
    }
    pub fn value(&self) -> f64 {
        let elapsed = self.start_time.elapsed().as_secs_f64();
        let total = self.duration.as_secs_f64();
        let t = (elapsed / total).clamp(0.0, 1.0);
        let eased = (self.easing)(t);
        self.from + (self.to - self.from) * eased
    }
    pub fn is_complete(&self) -> bool {
        self.start_time.elapsed() >= self.duration
    }
}
```

Add `pub mod animation;` to lib.rs.

**text-align center (render_style.rs):**
Find where widget text content is rendered. The text-align value is stored in ComputedStyle but likely not used during rendering. In render_style.rs or a helper function, add:

```rust
pub fn align_text(text: &str, width: usize, align: TextAlign) -> String {
    let text_width = text.chars().count();
    match align {
        TextAlign::Left => text.to_string(),
        TextAlign::Center => {
            let pad = width.saturating_sub(text_width) / 2;
            format!("{:>width$}{}", "", text, width = pad)
        }
        TextAlign::Right => {
            let pad = width.saturating_sub(text_width);
            format!("{:>width$}{}", "", text, width = pad)
        }
    }
}
```

Widgets that render text (Label, Button) should call this helper when rendering their content. Update Label::render and Button::render to use `align_text` with the computed text_align from their style.

**Worker progress (worker.rs):**
Add a WorkerProgress message type:

```rust
pub struct WorkerProgress<T: Send + 'static> {
    pub source_id: WidgetId,
    pub progress: T,
}
```

Add a `run_worker_with_progress` method to AppContext that takes a future AND a progress channel:

```rust
pub fn run_worker_with_progress<T, P>(
    &self,
    source_id: WidgetId,
    progress_fn: impl FnOnce(flume::Sender<P>) -> Pin<Box<dyn Future<Output = T>>> + 'static,
) -> tokio::task::AbortHandle
where
    T: Send + 'static,
    P: Send + 'static,
{
    let worker_tx = self.worker_tx.clone().expect("worker_tx not initialized");
    let progress_tx_for_worker = worker_tx.clone();
    let (progress_sender, progress_receiver) = flume::unbounded::<P>();

    // Spawn progress forwarding task
    let sid = source_id;
    let ptx = progress_tx_for_worker;
    tokio::task::spawn_local(async move {
        while let Ok(p) = progress_receiver.recv_async().await {
            let _ = ptx.send((sid, Box::new(WorkerProgress { source_id: sid, progress: p })));
        }
    });

    // Spawn main worker
    let fut = progress_fn(progress_sender);
    self.run_worker(source_id, fut)
}
```

Widgets receive WorkerProgress<T> messages via on_event, same as WorkerResult<T>.

**CSS hot-reload (app.rs):**
Add a file watcher for .tcss files. Use `notify` crate — BUT to avoid adding a new dependency, use a simpler polling approach:

Add a `css_watch_paths: Vec<PathBuf>` field to App. Add a builder method:
```rust
pub fn with_css_file(mut self, path: impl AsRef<Path>) -> Self
```
that reads the file, parses CSS, stores the stylesheet AND records the path + last modified time.

In the event loop, add a periodic check (every 2 seconds) using `tokio::time::interval(Duration::from_secs(2))`. On tick:
1. For each watched path, check `fs::metadata(path).modified()`.
2. If changed since last check, re-read, re-parse, replace the stylesheet in self.stylesheets.
3. Set needs_full_sync = true and trigger a render.

Store watched files as:
```rust
struct WatchedCss {
    path: PathBuf,
    last_modified: SystemTime,
    stylesheet_index: usize, // index into self.stylesheets
}
```

This is a simple polling watcher — no external dependency needed. 2-second poll interval is responsive enough for development.
