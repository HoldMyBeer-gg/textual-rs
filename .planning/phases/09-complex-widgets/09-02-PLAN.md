    **DirectoryTree struct:**
    ```rust
    pub struct DirectoryTree {
        pub root_path: PathBuf,
        pub show_hidden: bool,
        inner: RefCell<Tree>,
        loaded_paths: RefCell<HashSet<PathBuf>>,
        loading_paths: RefCell<HashSet<PathBuf>>,
        /// Canonical paths we've visited to detect cycles (NTFS junctions, symlink loops)
        visited_canonical: RefCell<HashSet<PathBuf>>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    **Constructor `::new(root: impl Into<PathBuf>)`:**
    1. Create a TreeNode with label = root directory name (last path component)
    2. Store root_path, create inner Tree with that root node
    3. show_hidden defaults to false
    4. Canonicalize root_path and insert into visited_canonical

    **`::with_show_hidden(mut self, show: bool) -> Self`** builder method.

    **Messages module:**
    ```rust
    pub mod messages {
        use crate::event::message::Message;
        use std::path::PathBuf;

        pub struct FileSelected { pub path: PathBuf }
        impl Message for FileSelected {}

        pub struct DirectorySelected { pub path: PathBuf }
        impl Message for DirectorySelected {}
    }
    ```

    **Helper: `fn read_directory(path: &Path, show_hidden: bool) -> Vec<DirEntryInfo>`**
    - Use `std::fs::read_dir(path)` (one level only, NOT walkdir â€” lazy loading per D-11)
    - For each entry, use `entry.path().symlink_metadata()` (per D-14, NOT `metadata()`)
    - Filter hidden files: if !show_hidden, skip entries whose name starts with '.' on Unix, or have hidden attribute on Windows (use `#[cfg(windows)]` with `std::os::windows::fs::MetadataExt::file_attributes` checking FILE_ATTRIBUTE_HIDDEN = 0x2)
    - Detect symlinks: `meta.file_type().is_symlink()` (per D-14)
    - Sort: directories first, then alphabetical by name (case-insensitive)
    - Return `Vec<DirEntryInfo>`

    **Cycle detection (per D-16 research, NTFS junction safety):**
    - Before spawning a worker for a directory path, canonicalize it via `std::fs::canonicalize(&path)`
    - If canonical path is already in `visited_canonical`, mark node as "(cycle)" and do NOT spawn worker
    - On successful load, add canonical path to visited_canonical

    **Worker dispatch in on_action("toggle"):**
    - When Tree emits NodeExpanded with a path, look up the TreeNode at that path
    - Extract the directory's PathBuf from TreeNode.data (store as `data: Some(path.to_string_lossy().to_string())`)
    - Guard: `if self.loaded_paths.borrow().contains(&dir_path) { return; }` (per D-11, cached children)
    - Guard: `if self.loading_paths.borrow().contains(&dir_path) { return; }` (prevent double-load)
    - Cycle check: canonicalize dir_path, check visited_canonical. If cycle, set node label to "(cycle)" and return.
    - Add dir_path to loading_paths
    - Insert a placeholder child node with label "Loading..." (per D-12) â€” set it as the only child of the expanded node
    - Spawn worker: `ctx.run_worker(id, async move { read_directory(&dir_path, show_hidden) })`

    **WorkerResult handling in on_event:**
    - Receive `WorkerResult<Vec<DirEntryInfo>>`
    - Find the tree node that was being loaded (match against loading_paths)
    - Remove placeholder "Loading..." child
    - For each DirEntryInfo:
      - Create TreeNode with label = name (+ "@" suffix if is_symlink, per D-15)
      - Set data = Some(path string)
      - If is_dir and NOT is_symlink: mark as expandable (has_children hint via adding an empty placeholder child that gets replaced on expand)
      - If is_symlink: do NOT add children â€” node is a leaf (per D-15, symlinks are not expanded)
    - Add canonical path to visited_canonical
    - Move dir_path from loading_paths to loaded_paths
    - Mark inner tree dirty for re-render

    **Widget trait delegation:**
    - `widget_type_name()` returns `"DirectoryTree"`
    - `on_mount(id)` â€” set own_id; also trigger initial load of root directory children (spawn worker for root_path)
    - `on_unmount(id)` â€” clear own_id
    - `key_bindings()` â€” delegate to inner Tree's bindings
    - `on_event(event, ctx)` â€” first check for WorkerResult<Vec<DirEntryInfo>>, then forward to inner Tree
    - `on_action(action, ctx)` â€” intercept "toggle" to check if lazy load needed, then delegate to inner Tree. On "select" (Enter), emit `messages::FileSelected` or `messages::DirectorySelected` based on node type.
    - `render(ctx, area, buf)` â€” delegate to `self.inner.borrow().render(ctx, area, buf)`. Apply color styling: directory labels in `$primary` color, file labels in `$text` color (per D-09). Use `â–¶`/`â–Ľ` indicators (per D-10 â€” already the Tree widget pattern).
    - `compose()` â€” returns empty vec (Tree is an inner field, not a mounted child)

    **Directory vs file color (per D-09):**
    In the Tree's render, directory labels should use the theme's primary color and file labels use text color. Since Tree renders its own entries, DirectoryTree should set node labels with color markers or override Tree's render. Simplest approach: prefix directory labels with a distinguishing marker in the TreeNode label and handle color in a custom render override. OR: store is_dir in TreeNode.data as a flag and have DirectoryTree's render apply colors when painting labels.
