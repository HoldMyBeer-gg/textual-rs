    **ToastSeverity enum:**

    pub enum ToastSeverity {
        Info,
        Warning,
        Error,
    }

    **ToastEntry struct:**
    ```rust
    #[derive(Debug, Clone)]
    pub struct ToastEntry {
        pub message: String,
        pub severity: ToastSeverity,
        pub timeout_ms: u64,
        pub elapsed_ticks: u32,
    }
    ```

    **`pub fn push_toast(toasts: &mut Vec<ToastEntry>, message: String, severity: ToastSeverity, timeout_ms: u64)`**
    - If `toasts.len() >= 5`, remove index 0 (drop oldest, per D-22)
    - Push new `ToastEntry { message, severity, timeout_ms, elapsed_ticks: 0 }`

    **`pub fn tick_toasts(toasts: &mut Vec<ToastEntry>)`**
    - For each toast, if timeout_ms > 0, increment elapsed_ticks by 1
    - Retain only toasts where `timeout_ms == 0 || (elapsed_ticks as u64 * 33) < timeout_ms`
    - (33ms per tick at ~30fps, per D-24)

    **`pub fn render_toasts(toasts: &[ToastEntry], area: Rect, buf: &mut Buffer, theme: &crate::css::theme::Theme)`**
    - If toasts is empty, return immediately
    - Toast dimensions: width = min(42, area.width.saturating_sub(2)), height per toast = 3 (1 border top + 1 content + 1 border bottom)
    - Position: right-aligned at `area.right() - toast_width - 1`, stacking upward from `area.bottom() - 1`
    - Iterate `toasts.iter().rev().enumerate()`:
      - `row_bottom = area.bottom().saturating_sub((i as u16 + 1) * 3)`
      - If row_bottom < area.y, break (no room for more toasts)
      - Draw a box border (thin/rounded) around the toast area
      - Fill interior with severity background color:
        - Info: resolve `$primary` from theme (use `theme.primary` RGB tuple) — render as dim background tint
        - Warning: resolve `$warning` from theme
        - Error: resolve `$error` from theme
      - Write message text inside the box, truncated to toast_width - 4 chars if longer
      - Severity indicator: first char of interior could be a symbol — `i` for info, `!` for warning, `x` for error — in severity color. OR just color the entire border in severity color (cleaner approach per D-09 "no emoji" principle).
    - Border color = severity color (Info: theme primary, Warning: theme warning, Error: theme error)
    - Text color = white/foreground on severity-tinted background

    **Rendering details:**
    - Use `ratatui::widgets::Block::bordered()` with `border_style` set to severity color, OR manually paint border chars and fill — follow the pattern from `paint_chrome` in app.rs or `draw_loading_spinner_overlay` in loading_indicator.rs for direct buffer writes.
    - The toast renders to `buf: &mut Buffer` directly — this is NOT a Widget in the arena. It's a paint function called from `render_widget_tree`.

    Include `#[cfg(test)] mod tests` with all behavior tests listed above. Use `ratatui::buffer::Buffer::empty(Rect::new(0, 0, 80, 24))` for render tests.

    /// Display a toast notification. Severity: Info/Warning/Error.
    /// timeout_ms: auto-dismiss after N ms (0 = persistent, no auto-dismiss).
    /// Default timeout: 3000ms.
    pub fn toast(&self, message: impl Into<String>, severity: ToastSeverity, timeout_ms: u64) {

    ```rust
    // Render toast notifications (below active_overlay so CommandPalette wins z-order)
    {
        let toasts = ctx.toast_entries.borrow();
        if !toasts.is_empty() {
            crate::widget::toast::render_toasts(&toasts, buf_area, frame.buffer_mut(), &ctx.theme);
        }

    // Paint the floating overlay (context menu, etc.) last, on top of everything
    if let Some(ref overlay) = *ctx.active_overlay.borrow() {

    }
    ```
