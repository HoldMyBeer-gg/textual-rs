  <action>
    Create `crates/textual-rs/src/widget/masked_input.rs` implementing the MaskedInput widget. Per D-01 through D-07:

    **MaskSlot enum:**
    ```rust
    enum MaskSlot {
        Digit,              // '#' — accepts c.is_ascii_digit()
        Letter,             // 'A' — accepts c.is_ascii_alphabetic()
        LetterOrBlank,      // 'a' — accepts c.is_ascii_alphabetic() || c == ' '
        Alphanumeric,       // 'N' — accepts c.is_alphanumeric()
        AlphaOrBlank,       // 'X' — accepts c.is_alphanumeric() || c == ' '
        Separator(char),    // any other char — literal, not editable
    }
    ```

    **Case modifiers:** Track `case_mode: Option<CaseMode>` (enum `Upper`/`Lower`) set by `>` and `<` in the mask string. These are NOT slots — they modify the next input slot's accepted character. When `>` is encountered during mask parsing, set case_mode to Upper for all subsequent slots until `<` is encountered (which sets Lower), or end of mask.

    **Struct:**
    ```rust
    pub struct MaskedInput {
        pub mask: String,
        raw_value: RefCell<String>,
        cursor_raw: Cell<usize>,
        mask_slots: Vec<MaskSlot>,
        /// For each input-slot index (0..N where N = number of non-separator slots),
        /// stores the display column where that slot renders.
        slot_display_cols: Vec<usize>,
        /// Case transform per input slot (None, Some(Upper), Some(Lower))
        slot_case: Vec<Option<CaseMode>>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    **Precompute in `::new(mask)`:**
    1. Parse mask string char-by-char into `mask_slots: Vec<MaskSlot>`
    2. Build `slot_display_cols: Vec<usize>` — for each non-separator slot, its column index in the full display string (counting separators). This is the O(1) lookup table from RESEARCH pitfall #1.
    3. Build `slot_case: Vec<Option<CaseMode>>` — case transform per input slot based on `>`/`<` modifiers.

    **Key methods:**
    - `pub fn value(&self) -> String` — returns `self.raw_value.borrow().clone()` (per D-03: raw chars only, no separators)
    - `fn build_display(&self) -> String` — walks mask_slots, interleaves raw_value chars at input-slot positions, fills unfilled slots with `_` placeholder, inserts separator chars as-is
    - `fn raw_pos_to_display_col(&self, raw_pos: usize) -> usize` — O(1) lookup into `slot_display_cols[raw_pos]`; if raw_pos >= slot_display_cols.len(), return last col + 1
    - `fn max_raw_len(&self) -> usize` — count of non-separator slots (max chars user can type)
    - `fn accepts_char(&self, slot_idx: usize, c: char) -> Option<char>` — checks MaskSlot at slot_idx, applies case transform from slot_case, returns Some(transformed_char) or None

    **Widget trait impl:**
    - `widget_type_name()` returns `"MaskedInput"`
    - `on_mount(id)` / `on_unmount(id)` — set/clear `own_id`
    - `is_focusable()` returns `true` (add this — Input does it via key_bindings presence)
    - `key_bindings()` — same bindings as Input: Left=cursor_left, Right=cursor_right, Backspace=delete_back, Enter=submit, Home=cursor_home, End=cursor_end
    - `on_event` — handle KeyCode::Char(c): look up current slot from cursor_raw, call accepts_char, if Some insert into raw_value at cursor_raw, advance cursor_raw, emit Changed. If None, silently ignore (per D-05).
    - `on_action("delete_back")` — if cursor_raw > 0: remove char at cursor_raw-1 from raw_value, decrement cursor_raw. Per D-06: this only ever removes a user-typed char since raw_value has no separators.
    - `on_action("cursor_left")` — if cursor_raw > 0: decrement cursor_raw by 1. Per D-07: since cursor is in raw space, separator skipping is automatic (display derivation handles it).
    - `on_action("cursor_right")` — if cursor_raw < raw_value.len(): increment cursor_raw by 1.
    - `on_action("cursor_home")` — set cursor_raw to 0.
    - `on_action("cursor_end")` — set cursor_raw to raw_value.len().
    - `on_action("submit")` — emit `messages::Submitted { value: self.value() }`.
    - `render()` — build display string via `build_display()`, compute cursor display col via `raw_pos_to_display_col(cursor_raw)`, render display string with ratatui styling. Show cursor character with `Modifier::REVERSED` when focused (same pattern as Input). Use `$text` color for the display string, `$text-muted` or dim for unfilled `_` placeholders.

    **Messages module:**
    ```rust
    pub mod messages {
        use crate::event::message::Message;
        pub struct Changed { pub value: String }
        impl Message for Changed {}
        pub struct Submitted { pub value: String }
        impl Message for Submitted {}
    }
    ```

    Emit `Changed` on every keystroke (discretion: matches Python Textual behavior — user gets live updates).

    Include `#[cfg(test)] mod tests { ... }` at bottom with all behavior tests listed above.
