The arboard crate requires an X11 display on Linux; if any test constructs `arboard::Clipboard::new()` directly, it will panic on ubuntu-latest CI (no DISPLAY). As of research time, arboard usage is only in runtime widget event handlers (input.rs, text_area.rs) wrapped in error-swallowing closures -- NOT in test code. Confirm this is still the case.

- If NO test code constructs `arboard::Clipboard`: Safe. No action needed. Log confirmation.
- If any test DOES construct `arboard::Clipboard`: Add `#[cfg_attr(not(target_os = "windows"), ignore)]` to that test, OR add `xvfb-action` to the test job's ubuntu step in ci.yml before the cargo test step:
