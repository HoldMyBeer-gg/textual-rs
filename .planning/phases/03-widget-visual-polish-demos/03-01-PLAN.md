<task type="auto">
  <name>Task 1: Button 3D depth borders</name>
  <files>crates/textual-rs/src/widget/button.rs</files>
  <action>
Modify Button::render() to draw 3D depth borders before the centered label. The button already has `height: 3` and `border: inner` from CSS, but the 3D shading must be applied programmatically in render() since CSS borders don't support per-edge color differentiation.

Implementation:
1. After getting `base_style`, extract the background color from the buffer cell at (area.x, area.y) — this is the button's CSS background (e.g., #2a2a3e from demo.tcss).
2. Compute two shades using `crate::canvas::blend_color`:
   - `light_edge = blend_color(bg_color, Color::Rgb(255, 255, 255), 0.25)` — 25% toward white
   - `dark_edge = blend_color(bg_color, Color::Rgb(0, 0, 0), 0.35)` — 35% toward black
3. Draw the top row of the content area with `light_edge` foreground (use "▔" repeated, or simply set the bg of the top row to light_edge). Simplest approach: paint the top row cells with `bg = light_edge` and the bottom row cells with `bg = dark_edge`.
4. When `is_pressed` is true, swap: top row gets `dark_edge`, bottom row gets `light_edge` — creating the visual "press in" illusion.
5. The middle row (where the label sits) keeps the original bg color.

Specifically in render():
- After the early return for zero area, before setting the label string:
  - Extract bg: `let bg = buf.cell((area.x, area.y)).and_then(|c| c.style().bg).unwrap_or(Color::Rgb(42, 42, 62));`
  - Compute shades: `let light = crate::canvas::blend_color(bg, Color::Rgb(255, 255, 255), 0.25);` and `let dark = crate::canvas::blend_color(bg, Color::Rgb(0, 0, 0), 0.35);`
  - If area.height >= 3: paint top row (area.y) cells with bg=light (or dark if pressed), bottom row (area.y + area.height - 1) cells with bg=dark (or light if pressed).
  - Use a simple loop: `for x in area.x..area.x+area.width { if let Some(cell) = buf.cell_mut((x, row_y)) { cell.set_bg(shade); } }`

Keep the existing label centering and BOLD/REVERSED logic. The REVERSED modifier on press stays — it provides the label flash. The border shading is additive visual depth.

Add `use ratatui::style::Color;` at the top of render() if not already imported.

Modify DataTable::render() in the data rows section (around line 470) to apply alternating row background colors for zebra striping.

Implementation:
1. Before the data row loop, extract the table's background color from the buffer: `let table_bg = buf.cell((area.x, area.y)).and_then(|c| c.style().bg).unwrap_or(Color::Rgb(26, 26, 46));`
2. Compute the alternate row shade: `let alt_bg = crate::canvas::blend_color(table_bg, Color::Rgb(255, 255, 255), 0.06);` — a subtle 6% lighter stripe.
3. Inside the row loop, determine if this is a zebra row: `let is_alt_row = row_idx % 2 == 1;`
4. For alt rows that are NOT the cursor row, fill the entire row background BEFORE rendering cell text:
   ```rust
   if is_alt_row && !is_cursor_row {
       let stripe_bg = ratatui::style::Style::default().bg(alt_bg);
       for fill_x in area.x..area.x + area.width {
           if let Some(cell) = buf.cell_mut((fill_x, row_y)) {
               cell.set_style(stripe_bg);
           }
       }
   }
   ```
5. For alt rows, also apply the alt_bg to the row_style used for text: modify the non-cursor branch to check is_alt_row and set `.bg(alt_bg)` on the style.
6. The cursor row styling (accent color + BOLD) takes priority and should NOT get zebra striping — this is already handled by the `is_cursor_row` check.

Add `use crate::canvas;` import if not already present. The Color import is already available via `ratatui::style::Color`.
