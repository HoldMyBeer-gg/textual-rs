3. Add `push_screen_wait` method — this is an async method that returns a future:
```rust
/// Push a modal screen and await a typed result.
///
/// The pushed screen (or its inner widget) must call `pop_screen_with(value)`
/// to dismiss and deliver the result. The returned future resolves to `T`.
///
/// # Panics
/// Panics if the result type `T` does not match what pop_screen_with provides.
///
/// # Example
/// ```ignore
/// let confirmed: bool = ctx.push_screen_wait(
///     Box::new(ModalScreen::new(Box::new(ConfirmDialog)))
/// ).await;
/// ```
pub fn push_screen_wait<T: Any + Send + 'static>(
    &self,
    screen: Box<dyn Widget>,
) -> tokio::sync::oneshot::Receiver<Box<dyn Any>> {
    let (tx, rx) = tokio::sync::oneshot::channel();
    self.pending_screen_wait_pushes.borrow_mut().push((screen, tx));
    rx
}
```

Note: The caller will need to `.await` the receiver and downcast. We provide a convenience wrapper as a free function or leave it to the caller. The actual API pattern for widgets:
```rust
let rx = ctx.push_screen_wait::<bool>(Box::new(ModalScreen::new(Box::new(dialog))));
// Later, in an async context or via worker:
// let result = rx.await.unwrap();
// let confirmed: bool = *result.downcast::<bool>().unwrap();
```

However, since `on_action` is sync (`&self`), the typical usage is through `run_worker`:
```rust
// In on_action:
let rx = ctx.push_screen_wait::<bool>(screen);
ctx.run_worker(self_id, async move {
    let result = rx.await.unwrap();
    *result.downcast::<bool>().unwrap()
});
// Result arrives as WorkerResult<bool> in on_event
```

4. Add `pop_screen_with` method:
```rust
/// Pop the top screen and deliver a typed result to the push_screen_wait caller.
///
/// The value is stored and delivered when the deferred pop is processed.
/// Call this from on_action in a modal's inner widget.
pub fn pop_screen_with<T: Any + 'static>(&self, value: T) {
    self.pending_screen_results
        .borrow_mut()
        .insert(
            // We don't know our screen_id here — store with a sentinel.
            // process_deferred_screens will match it to the top screen being popped.
            WidgetId::default(), // placeholder — see app.rs changes
            Box::new(value),
        );
    self.pop_screen_deferred();
}
```

Actually, better approach — use a simpler mechanism: store the result value directly without needing to know the screen ID. Since pop_screen_with always pops the TOP screen, and there's at most one pop-with-result per event cycle, use a single-slot:
```rust
/// Result value for the next screen pop (set by pop_screen_with, consumed by process_deferred_screens).
pub pending_pop_result: RefCell<Option<Box<dyn Any>>>,
```

And `pop_screen_with`:
```rust
pub fn pop_screen_with<T: Any + 'static>(&self, value: T) {
    *self.pending_pop_result.borrow_mut() = Some(Box::new(value));
    self.pop_screen_deferred();
}
```

Initialize in new(): `pending_pop_result: RefCell::new(None),`

**App changes** (app.rs, `process_deferred_screens`):

1. After the existing push_screen_wait screens are pushed, record the oneshot senders:
```rust
// Process push_screen_wait requests
let wait_pushes: Vec<_> = self.ctx.pending_screen_wait_pushes.borrow_mut().drain(..).collect();
for (screen, sender) in wait_pushes {
    let screen_id = push_screen(screen, &mut self.ctx);
    self.ctx.screen_result_senders.borrow_mut().insert(screen_id, sender);
    self.needs_full_sync = true;
}
```

2. Modify the pop processing: when popping a screen, check if it has a result sender and a pending result:
```rust
let pops = self.ctx.pending_screen_pops.get();
if pops > 0 {
    self.ctx.pending_screen_pops.set(0);
    for _ in 0..pops {
        // Check for pop_screen_with result
        let result = self.ctx.pending_pop_result.borrow_mut().take();
        let top_screen_id = self.ctx.screen_stack.last().copied();

        pop_screen(&mut self.ctx);

        // If there's a result and a sender for this screen, deliver it
        if let (Some(result), Some(screen_id)) = (result, top_screen_id) {
            if let Some(sender) = self.ctx.screen_result_senders.borrow_mut().remove(&screen_id) {
                let _ = sender.send(result);
            }
        }
    }
    self.needs_full_sync = true;
}
```
