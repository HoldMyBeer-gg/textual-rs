File header comment:
```rust
// Tutorial 06: Screens — push/pop navigation and modal dialogs
//
// This tutorial shows how to:
//   1. Push a new screen onto the screen stack for navigation
//   2. Pop a screen to go back
//   3. Push a ModalScreen that blocks input to screens below
//   4. Use push_screen_wait + pop_screen_with for typed modal results
//
// Run with: cargo run --example tutorial_06_screens
// Quit with: q or Ctrl+C
```

**CSS** (const CSS: &str):
- `ScreenDemoScreen` (root): vertical layout, background: $background, color: $foreground
- `Header`: height 1, background: $panel, color: $primary
- `Footer`: height 1, background: $panel, color: $text
- `MainContent`: flex-grow 1, vertical layout, padding 1
- `NavScreen`: vertical layout, background: $surface, color: $foreground
- `ConfirmDialog`: vertical layout, width 50, height 8, background: $panel, border: heavy, padding 1, align-self: center (centered modal)
- Buttons: standard button styles

**Widgets**:

1. `ScreenDemoScreen` (root screen):
   - Composes: Header("Tutorial 06: Screens"), MainContent, Footer with key bindings
   - Key bindings: q = quit

2. `MainContent`:
   - Stores `own_id: Cell<Option<WidgetId>>`, `result_label: Reactive<String>`
   - Composes:
     - Label "Screen Stack Demo" (title)
     - Label showing `result_label` reactive value (initially "No result yet")
     - Button "Push Navigation Screen" (action: push_nav)
     - Button "Push Modal Dialog" (action: push_modal)
   - `on_action`:
     - "push_nav": `ctx.push_screen_deferred(Box::new(NavScreen))`
     - "push_modal": push a ConfirmDialog via push_screen_wait:
       ```rust
       let rx = ctx.push_screen_wait::<bool>(
           Box::new(ModalScreen::new(Box::new(ConfirmDialog)))
       );
       ctx.run_worker(self_id, async move {
           match rx.await {
               Ok(result) => *result.downcast::<bool>().unwrap(),
               Err(_) => false,
           }
       });
       ```
   - `on_event`: handle `WorkerResult<bool>`:
     ```rust
     if let Some(wr) = event.downcast_ref::<WorkerResult<bool>>() {
         let msg = if wr.value { "You chose: OK" } else { "You chose: Cancel" };
         self.result_label.set(msg.to_string());
         return EventPropagation::Stop;
     }
     ```

3. `NavScreen` (non-modal, pushed for navigation):
   - Composes: Header("Navigation Screen"), Label "This is a pushed navigation screen", Button "Go Back" (action: go_back), Footer
   - `on_action`: "go_back" => `ctx.pop_screen_deferred()`
   - Shows that keyboard focus moves to NavScreen and back

4. `ConfirmDialog` (inner widget for ModalScreen):
   - Composes: Label "Are you sure?", Horizontal layout with Button "OK" and Button "Cancel"
   - `on_action`:
     - "ok" => `ctx.pop_screen_with(true)`
     - "cancel" => `ctx.pop_screen_with(false)`
   - Visual: centered dialog box with border, background visible and dimmed behind it
