  <name>Task 1: Multi-screen rendering with modal dimming</name>
  <files>crates/textual-rs/src/app.rs</files>
  <action>
Modify `full_render_pass` (line ~770) to render ALL screens in the stack, not just the top one. Per D-01.

Current code renders only the top screen:
```rust
let screen_id = match self.ctx.screen_stack.last().copied() { ... };
// ... cascade, layout, render for screen_id only
```

Change to iterate all screens bottom-up:
1. For each screen in `self.ctx.screen_stack.iter()` (bottom to top):
   a. Apply CSS cascade via `apply_cascade_to_tree(screen_id, ...)`
   b. Sync layout via `self.bridge.sync_dirty_subtree(screen_id, ...)`
   c. Compute layout via `self.bridge.compute_layout(screen_id, ...)`
   d. Clear dirty flags via `clear_dirty_subtree(screen_id, ...)`

2. Build hit map from TOP screen only (already correct — keep building from `screen_stack.last()`). This ensures mouse input is blocked to background screens (SC-3).

3. In the `terminal.draw(|frame| { ... })` closure, render each screen bottom-up:
   - For each `screen_id` in `self.ctx.screen_stack.iter()`:
     - Call `render_widget_tree(screen_id, ctx_ref, bridge_ref, frame)`
     - If this is NOT the last (top) screen AND the next screen up is modal (`ctx.arena.get(next_screen).map(|w| w.is_modal()).unwrap_or(false)`), apply dimming to the frame buffer AFTER rendering this screen's tree
   - Dimming implementation (per D-02, Claude's discretion): After rendering a background screen, iterate all cells in `frame.buffer_mut()` and darken them by blending toward black. For each cell with `Color::Rgb(r, g, b)` foreground/background, multiply each channel by 0.4 (i.e., `(r as f32 * 0.4) as u8`). For indexed/named colors, convert to approximate RGB first or apply a uniform dark tint. This gives a 60% darkening effect matching Python Textual's `$background 60%`.

4. Apply the same multi-screen rendering logic to `render_to_test_backend` (line ~823). This method creates its own Terminal and draws — apply the same bottom-up iteration with dimming.

5. Toast and overlay rendering (lines 1329-1341) remain after all screen renders — they should already be correct since they render on top of everything.

Key constraint: The `needs_full_sync` flag should be set to `true` after any screen push/pop to ensure all screens get a full layout sync. Add `self.needs_full_sync = true;` inside `process_deferred_screens` after any push or pop occurs.

Keyboard input already scoped to top screen: `handle_key_event` dispatches to `self.ctx.focused_widget` which is always on the top screen (push_screen calls advance_focus on new screen, pop_screen restores). Tab cycling uses `screen_stack.last()` as root. No changes needed for keyboard blocking (SC-3 keyboard part).

Mouse input already scoped to top screen: `handle_mouse_event` uses `self.hit_map` which is built from top screen only. No changes needed for mouse blocking (SC-3 mouse part).

Clean repaint after pop (SC-4): After pop_screen, the background screen is now the top screen. On next render, full_render_pass renders it normally. Since the terminal buffer is fully redrawn each frame (ratatui clears and redraws), there are no artifacts. The `needs_full_sync = true` flag ensures layout is fresh.
