# Phase 05 Plan 01: Screen Stack — Multi-Screen Rendering and Demo

## Objective

Complete the screen stack feature so that:
1. Modal screens render visually over the background screen (multi-screen layered rendering)
2. Input is correctly scoped to the top screen (keyboard focus + mouse hit-test)
3. Integration tests verify push/pop/modal/focus-restore behavior end-to-end
4. A working demo example shows screen navigation

## Context

The infrastructure (push_screen, pop_screen, focus_history, ModalScreen, push_screen_deferred,
pop_screen_deferred) is already implemented in:
- `crates/textual-rs/src/widget/tree.rs` — push_screen / pop_screen / advance_focus
- `crates/textual-rs/src/widget/context.rs` — AppContext fields + deferred API
- `crates/textual-rs/src/widget/screen.rs` — ModalScreen struct
- `crates/textual-rs/src/app.rs` — process_deferred_screens + full_render_pass

Gap: `full_render_pass` renders only the top screen (`screen_stack.last()`). When a ModalScreen
is pushed, the background screen is not painted — the modal appears on a blank frame. The fix is
to render ALL screens from bottom to top so modal dialogs visually overlay the background.

## Tasks

### Task 1: Multi-Screen Layered Rendering
`type="auto"`

Modify `full_render_pass` in `app.rs` to render every screen in the stack from bottom to top.

Behavior:
- All screens below the top are rendered first, then the top screen is rendered over them
- CSS cascade, layout, and dirty clearing apply only to the **top** screen (performance: no
  re-layout of frozen background screens on every frame)
- The hit map is built from the **top** screen only (ensures mouse is blocked from background)
- The render order is: screen_stack[0], screen_stack[1], ..., screen_stack[last]

### Task 2: Integration Tests for Screen Stack
`type="auto"`

Add tests in `app.rs` (or a new `tests/screen_stack.rs`) that verify:
- push_screen → keyboard input dispatched to top screen only
- pop_screen → background screen repaint (no blank frame)
- ModalScreen → mouse hit-test blocked from background screen
- Focus restore after pop

Use TestApp / render_to_test_backend for rendering assertions.

### Task 3: Screen Stack Demo Example
`type="auto"`

Create `crates/textual-rs/examples/screen_stack.rs` demonstrating:
- Main screen with a "Push Screen" button and "Open Modal" button
- Pressing "Push Screen" pushes a second screen; pressing Back returns
- Pressing "Open Modal" pushes a ModalScreen dialog centered on screen
- Modal dialog has OK/Cancel buttons that pop the modal
- Status label on main screen shows which screen is active

## Verification

- `cargo test -p textual-rs screen_stack` passes
- `cargo build --example screen_stack` succeeds
- All existing tests still pass: `cargo test -p textual-rs`

## Success Criteria

1. `full_render_pass` renders all screens bottom-to-top
2. Modal screens show background content underneath
3. Integration tests for push/pop/modal pass
4. Demo example compiles and runs
