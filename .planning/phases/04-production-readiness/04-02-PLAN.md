**CSS types (types.rs):**
Add new variant to TcssValue:
```rust
BorderWithVariable(BorderStyle, String),
```

**Property parser (property.rs):**
In the "border" arm of parse_property_value, after parsing the border style ident and before trying parse_color, try `try_parse_variable(input)`. If a variable is found, return `TcssValue::BorderWithVariable(style, var_name)`. If not, fall through to existing parse_color logic.

```rust
"border" => {
    let name = input.expect_ident_cloned()...;
    let style = match name.as_ref() { ... };
    // Try variable first
    if let Some(var_name) = try_parse_variable(input) {
        return Ok(Some(TcssValue::BorderWithVariable(style, var_name)));
    }
    // Then try literal color
    let color = parse_color(input).ok();
    ...
}
```

**Cascade (cascade.rs):**
In resolve_variables, add a case for BorderWithVariable:
```rust
TcssValue::BorderWithVariable(ref style, ref name) => {
    if let Some(color) = theme.resolve(name) {
        Declaration {
            property: d.property.clone(),
            value: TcssValue::BorderWithColor(*style, color),
        }
    } else {
        d.clone() // unresolved variable left as-is
    }
}
```

**McGugan Box fallback (canvas.rs):**
The RIGHT_ONE_QUARTER character U+1FB87 is from Unicode 13 "Legacy Computing Supplement" and is not available in many terminal fonts. Add a fallback mechanism:

Add a module-level function `right_border_char() -> &'static str` that returns RIGHT_ONE_QUARTER. Also add a constant `RIGHT_BORDER_FALLBACK: &str = "\u{2595}"` (U+2595 RIGHT ONE EIGHTH BLOCK, widely supported since Unicode 1.1).

In mcgugan_box(), use RIGHT_BORDER_FALLBACK instead of RIGHT_ONE_QUARTER for the right edge. U+2595 is thinner but universally available. This is the pragmatic choice — Python Textual also uses U+2595 for broad compatibility.

Add tests for the new border+variable parsing.
