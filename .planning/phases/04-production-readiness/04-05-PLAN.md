**Syntax highlighting in Markdown code blocks:**
Currently markdown.rs renders code blocks with a dark background but no syntax highlighting. Add basic keyword highlighting:

Create a simple highlighter function (no external dependency — avoid adding syntect which is 5MB+):

```rust
fn highlight_code(code: &str, language: &str) -> Vec<(String, Style)> {
    // Simple keyword-based highlighting for common languages
    let keywords = match language {
        "rust" => &["fn", "let", "mut", "pub", "struct", "enum", "impl", "use", "mod", "match", "if", "else", "for", "while", "return", "self", "Self", "async", "await", "trait", "where", "type", "const", "static"][..],
        "python" => &["def", "class", "import", "from", "return", "if", "else", "elif", "for", "while", "with", "as", "try", "except", "raise", "yield", "async", "await"][..],
        "javascript" | "js" | "typescript" | "ts" => &["function", "const", "let", "var", "return", "if", "else", "for", "while", "class", "import", "export", "from", "async", "await", "new", "this"][..],
        _ => &[][..],
    };
    // Tokenize by whitespace/punctuation, color keywords in accent, strings in green, comments in gray
    // Return vec of (text, style) spans
}
```

In Markdown::render, when rendering a code block, use highlight_code to produce colored spans instead of monochrome text. Use theme colors:
- Keywords: accent color (bold)
- Strings (detected by quote delimiters): success color
- Comments (// or # prefix): muted gray
- Default: foreground color
