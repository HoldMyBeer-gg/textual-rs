<task type="auto">
  <name>Task 1: Terminal capability detection</name>
  <files>crates/textual-rs/src/terminal.rs</files>
  <action>
Add a `TerminalCaps` struct to terminal.rs:

```rust
#[derive(Debug, Clone)]
pub struct TerminalCaps {
    pub color_depth: ColorDepth,
    pub unicode: bool,
    pub mouse: bool,
    pub title: bool,
}

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum ColorDepth {
    /// No color (dumb terminal)
    NoColor,
    /// 16 standard colors
    Standard,
    /// 256 color palette
    EightBit,
    /// 24-bit true color (16M colors)
    TrueColor,
}
```

Implement `TerminalCaps::detect() -> Self`:

**Color depth detection:**
1. Check `COLORTERM` env var: if "truecolor" or "24bit" -> TrueColor
2. Check `TERM` env var: if contains "256color" -> EightBit
3. On Windows: Windows Terminal and modern conhost support TrueColor. Check `WT_SESSION` env var (Windows Terminal sets this). If present -> TrueColor. Otherwise -> EightBit (Windows 10+ conhost supports 256).
4. Fallback: Standard (16 colors)

**Unicode detection:**
1. Check `LANG` or `LC_ALL` env vars for "UTF-8" or "utf8" -> true
2. On Windows: assume true (Windows Terminal and modern conhost handle Unicode)
3. Check `TERM` for "xterm" family -> true
4. Fallback: false

**Mouse:** Always true (crossterm enables it)
**Title:** Check `TERM` not "dumb" and not "linux" (Linux console doesn't support title)

Add `pub fn detect_capabilities() -> TerminalCaps` as a module-level convenience function.

This is purely informational for now — the app stores it but doesn't downgrade rendering yet. Future plans can use it to select ASCII-only borders, 256-color palette fallbacks, etc.

**Verify resize reflow:**
The current resize handler at line 435 already calls `self.full_render_pass(&mut terminal)?` which recomputes layout via `bridge.compute_layout(screen_id, size.width, size.height, &self.ctx)`. This correctly uses the NEW terminal size from `terminal.size()`.

However, ensure the bridge does a FULL sync on resize (not just dirty sync), because resize may change which widgets are visible, scrollbar positions, etc. In the Resize handler, set `self.needs_full_sync = true` before calling full_render_pass:

```rust
Ok(AppEvent::Resize(_, _)) => {
    self.needs_full_sync = true;
    self.full_render_pass(&mut terminal)?;
}
```

This forces bridge.sync_subtree (full tree walk) instead of sync_dirty_subtree on the next render pass after resize, ensuring all widgets get proper layout recomputation.
